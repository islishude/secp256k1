## Result

The opt-in `secp256k1_asm` backend now uses a signed W6 packed fixed-base table
and meets the 18,000 ns recoverable-signing target. The retained ten-run median
is 16,806.5 ns/op with 0 B/op and 0 allocs/op. The default backend remains W5.

An adjacent baseline/candidate default verification run removed thermal drift
from an earlier non-adjacent sample. Default hot verification changed by
+0.53%, compressed cold verification by +0.95%, and uncompressed cold
verification by -4.30%; all retained gates passed. The default ten-run signing
and fixed-base samples were also non-regressive at -0.29% and -3.69%.

## Implementation

The generator now writes shared wNAF, endomorphism, and width-7 verification
comb data separately from the fixed-base table. W5 is built under
`!arm64 || !secp256k1_asm`; W6 is built under `arm64 && secp256k1_asm`. Each
generated artifact records its parameters, generator-source hash, and data
hash. A second `go generate ./...` produced identical SHA-256 hashes for all
three files.

The tagged path performs 43 fixed signed windows. Each selection reads all 32
packed affine points through a monotonically incremented pointer. Disassembly
of the 1,024-byte selector contains 32 `VLD1`, 31 `CMP`, 31 `CSETM`, and no
branch. The secret magnitude is used only by compare/mask instructions. Zero
digits and the first-window infinity case remain mask-selected, and subsequent
windows use the existing complete mixed addition and multiplication-by-21
path. The remaining branches in the Go-path disassembly trace to public window
indices, fixed word-boundary conditions, or fixed-size clearing loops; the
complete mixed-add function itself has only its stack-growth guard.

The final 5-second signing profile measured 17,144 ns/op. Fixed-base
multiplication accounted for 47.43% cumulative CPU, complete mixed addition
39.33%, affine conversion 22.53%, and the W6 selector 4.35% flat. Per the plan,
W6 reached the target, so no fused complete-mixed-add candidate was started.

## Validation

- Generated W5/W6 tables are compared point-by-point with dynamic builders in
  their respective default/tagged builds.
- W6 selection covers all 43 windows and magnitudes 0 through 32. A dedicated
  recoding test checks the highest window, incoming carry, terminal carry, the
  magnitude bound, and exact reconstruction of `2^256-1`.
- Production W6, dynamic W6, dynamic W5, and W4 oracle results match for zero,
  one, two, the bit-255 boundary, `n-1`, and 1,000 deterministic random scalars.
- Existing compact/recoverable signature goldens pass in default and tagged
  builds.
- Default/tagged non-cached race tests, vet, golangci-lint, two 10-second fuzz
  smokes, constant-time smokes, dependency and variable-time audits,
  generation reproducibility, benchmark-module tests, and Linux ARM64/AMD64
  cross-builds passed.
