# ARM64 Montgomery and adaptive comb results

Date: 2026-07-10  
Go: go1.26.4  
GOOS/GOARCH: darwin/arm64  
CPU: Apple M3 Max  
Base revision: working tree after the initial performance plan

## Design

The optional `secp256k1_asm` backend preserves the existing saturated 4x64
Montgomery representation. Its ARM64 field operations are fixed instruction
sequences with no secret-dependent branches or memory addresses. The backend
remains opt-in pending independent review.

Verification now keeps the existing wNAF/GLV tables for the first eight valid
signatures under a public key. On the ninth verification it builds a 7-bit
fixed-base comb table once and reuses it. This preserves the cold single-use
path while reducing the repeated-key double-scalar loop from roughly 128
doublings to 37. Comb digits and table lookups depend only on public inputs.

The generator comb table is generated and checked in with the existing static
precomputation artifacts. A repeatedly used public key retains its original
wNAF tables and adds a 9,216-byte comb table; keys used at most eight times do
not allocate it.

## ARM64 opt-in result

Command:

```text
go test -tags=secp256k1_asm -run '^$' \
  -bench '^(BenchmarkVerifyHotPublicKey|BenchmarkVerifyParseCompressedCold|BenchmarkVerifyParseUncompressedCold|BenchmarkDoubleScalarBaseMultCombVartime)$' \
  -benchmem -count=10 .
go test -tags=secp256k1_asm -run '^$' \
  -bench '^BenchmarkSignRecoverable$' \
  -benchmem -count=10 .
```

| Benchmark | Previous median | New median | Delta | Allocations |
| --- | ---: | ---: | ---: | ---: |
| VerifyHotPublicKey | 30,143 ns | 19,370 ns | -35.7% | 0 B, 0 allocs |
| VerifyParseCompressedCold | 64,415 ns | 58,259 ns | -9.6% | 9,472 B, 1 alloc |
| VerifyParseUncompressedCold | 60,053 ns | 56,241 ns | -6.3% | 9,472 B, 1 alloc |
| SignRecoverable | 24,632 ns | 23,264 ns | -5.6% | 0 B, 0 allocs |
| DoubleScalarBaseMultCombVartime | n/a | 17,031 ns | n/a | 0 B, 0 allocs |

The 23,000 ns verification and 23,500 ns recoverable-signing goals are both
met on the target machine.

## Default pure-Go result

The adaptive comb verification path is architecture-independent. Without the
assembly tag, the ten-run medians were 20,651 ns for hot verification, 64,868
ns for compressed cold verification, 60,374 ns for uncompressed cold
verification, and 24,694 ns for recoverable signing. The default build remains
fully functional and keeps its existing fiat arithmetic backend.

## Arithmetic gates

Against the fiat microbenchmarks on this machine, the retained ARM64 field
operations measured approximately:

- Square: 10.04 ns to 7.31 ns, about 27% faster.
- Add: 2.06 ns to 1.91 ns, about 7% faster.

The Mul and Sub assembly candidates improved their microbenchmarks by only
about 3% and 2%, respectively, so they were removed. Tagged builds continue to
use the generated fiat implementations for those operations.

No scalar-field assembly was added in this phase because field squaring plus
comb already met its signing target. The subsequent
[`20260711-arm64-sign-sprint.md`](20260711-arm64-sign-sprint.md) phase added
register-resident scalar/base-field repeated squaring and constant-time W5
selection, while keeping default enablement blocked on an independent audit.
