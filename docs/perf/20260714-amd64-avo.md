## Result

The opt-in `secp256k1_asm` AMD64 backend retains two ADX/BMI2 base-field
kernels, Mul and Square, plus a fixed-length SSE2 selector for the shared
signed-W6 base-point table. Both `GOAMD64=v1` and `v3` exceed the final 10%
recoverable-signing and hot-verification gates with 0 B/op and 0 allocs/op.

Each row compares default and tagged binaries in alternating order ten times
within one Ubuntu job. Absolute values from different jobs are not compared.
The v1 job ran on an AMD EPYC 9V74 and the v3 job on an Intel Xeon 6973P-C;
both reported ADX+BMI2 through the generated CPUID helper.

| --- | --- | ---: | ---: | ---: | ---: | --- |
| v1 | SignRecoverable | 51,828.0 ns | 38,853.5 ns | -25.03% | 0 B / 0 | pass: >=10% |
| v1 | VerifyHotPublicKey | 43,085.5 ns | 35,502.0 ns | -17.60% | 0 B / 0 | pass: >=10% |
| v3 | SignRecoverable | 30,050.0 ns | 24,078.5 ns | -19.87% | 0 B / 0 | pass: >=10% |
| v3 | VerifyHotPublicKey | 23,915.0 ns | 21,390.5 ns | -10.56% | 0 B / 0 | pass: >=10% |

Every tracked signing, verification, recovery, and public-key workload
improved. Depending on the CPU and GOAMD64 level, fixed-base multiplication
improved 33.91-40.89%, public-key derivation 18.65-24.35%, recovery
9.80-16.76%, digest signing 18.19-24.69%, digest verification 9.65-17.39%,
and cold verification 8.45-17.82%. Root ScalarInv and ScalarInvVartime changed
by at most 0.58%.

SquareN remain on fiat and are still tracked, but have no AMD64 assembly
symbols or production routing.

The MulByB3 and SquareN fallback sources are identical to the default fiat

## Rejected candidates and stopping decisions

The first generated field candidate stored Montgomery accumulator state on the
stack. It was deleted after Mul improved only 1.84% at v1 and 2.88% at v3;
signing improved 7.80%/5.65% and hot verification 2.68%/2.22%.

The register-resident SquareN candidate improved its microbenchmark by
33.90%/33.39%, but independently regressed hot verification by 1.25% at v1 and
1.07% at v3. The specialized multiply-by-21 candidate improved 73.33%/72.03%,
but regressed v1 hot verification by 1.09%. Both candidates were removed from
the generator, stubs, assembly, routing, benchmark selector, and disassembly
audit. Their rejection evidence is in
[run 29340639169](https://github.com/islishude/secp256k1/actions/runs/29340639169).

Add/Sub were not attempted because retained field Mul/Square already pushed
hot verification past 10%. Scalar assembly was not attempted because W6 pushed
signing past 10%. No fused point arithmetic, W7, API changes, or new
variable-time secret path was introduced.

## Implementation and control flow

The separate `asm` module pins
[`github.com/mmcloughlin/avo v0.6.0`](https://github.com/mmcloughlin/avo#readme);
avo and its generator-only dependencies do not enter the root module.
Generated CPUID code checks the maximum basic leaf and caches leaf-7 BMI2 bit 8
plus ADX bit 19. `GOAMD64` is deliberately not used as an ADX proxy because
[the v3 requirements](https://go.dev/wiki/MinimumRequirements#amd64) include
BMI2 but not ADX. Tagged CPUs without both features call fiat.

Mul stages all four limbs of both operands before arithmetic, making `out == x`,
`out == y`, and `x == y` explicit. The first operand occupies four general
registers and the second occupies two SSE2 registers, so Mul has no stack
frame. Mul and Square use a five-limb, four-row CIOS Montgomery accumulator
specialized for
`p = 2^256 - 2^32 - 977`, followed by one fixed conditional subtraction. Each
symbol contains 20 `MULX`, 20 `ADCX`, 20 `ADOX`, four `CMOV`, and no branch.

The W6 selector is generated by avo using only baseline AMD64/SSE2
instructions. It is fully unrolled, reads all 32 consecutive 64-byte entries,
and uses the secret magnitude only in 31 compare-to-mask sequences. Native
disassembly contains 132 `MOVDQU` instructions (128 table reads plus four
stores), 31 `CMP`, 31 `SETE`, and no branch. The public Go recoding loop always
executes 43 windows.

| Symbol | Text size |
| --- | ---: |
| `mulMontgomeryADXAsm.abi0` | 872 bytes |
| `squareMontgomeryADXAsm.abi0` | 840 bytes |
| `selectGeneratorW6.abi0` | 3,193 bytes |

## Validation

- Mul, Square, fiat fallback operations, canonical output, and all alias forms
  were checked against fiat over carry/borrow edges and 100,000 deterministic
  random pairs. SquareN zero and generated-chain counts are also covered.
- W6 covers all 43 windows and magnitudes 0 through 32, highest-window carry,
  table regeneration, boundary scalars, and 1,000 deterministic random scalar
  comparisons against W4/W5/dynamic-W6 oracles. Existing signature goldens pass.
- Default and tagged full/race tests, default and tagged fuzzing, constant-time
  smoke, lint/vet, dependency/vartime audits, generator tidy/test/vet,
  generation consistency, and benchmark-module tests pass.
- Tagged AMD64 passes native Linux and Windows at v1/v3; Darwin AMD64 v1/v3
  cross-builds pass. GNU disassembly validates symbols, instructions, branches,
  ABI-facing entry points, and rejected-symbol absence.
- The root module still has no third-party dependencies, the default build
  remains pure Go/W5, and public APIs, RFC6979 behavior, and signature encodings
  are unchanged.
